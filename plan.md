# 원하는 페이지를 찾기 힘든 웹사이트 구현

## 구현의 목적

- 유저와의 대화를 통해 원하는 기능을 찾아주는 AI 챗봇 시연
- 사용하기 어려운 웹사이트의 사용을 AI 챗봇이 도와줄 수 있어야 함

## 웹사이트가 만족할 조건

- 각 페이지 메뉴가 많고 depth가 있어 유저가 원하는 액션을 수행할 페이지를 쉽게 찾기 어려움
- 페이지 내에서도 원하는 버튼을 찾으려면 스크롤 해야함
- 페이지 redirection & scroll을 통해 원하는 액션 버튼에 다다를 수 있어야 함
- query param 등을 이용한 자동 스크롤 지원이 필요

## 웹사이트 컨셉

- @channel.io/bezier-react의 디자인 시스템 활용
- 문서 검토 등이 포함된 ERP
- 메뉴가 많음 (모든 메뉴가 작동 가능할 필요는 없음)
- 대학교에서 사용할만한 웹사이트. 학생, 교수 등이 사용
